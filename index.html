CH 01

Spatial Resolution vs.
Intensity Resolution

Core Definitions
🔍
Spatial Resolution

Ability to distinguish fine spatial details. Determined by pixel density (how small a feature can be resolved). High spatial res → see thin lines, cracks, leaf veins, sharp boundaries.

🎨
Intensity Resolution (Quantization)

Number of distinct gray/color levels per pixel. Measured in bits per pixel (bpp). High intensity res → detect subtle color/brightness differences.

📐 Quantization LevelsNumber of Levels = 2^(bits per pixel) Examples: 4-bit → 2^4 = 16 levels 6-bit → 2^6 = 64 levels 8-bit → 2^8 = 256 levels 14-bit → 2^14 = 16,384 levels
Decision Framework
Task / ScenarioPrioritizeWhy
Detecting fine cracks, thin lines, leaf lesions, boundariesSpatialNeed to see small geometric features
Detecting subtle color/intensity shift (early disease, oil sheen, temperature)IntensityNeed fine gradient discrimination
Classifying surface materials by reflectance/contrastIntensityDifferent materials → different intensity bands
Tracking exact oil-water boundariesSpatialSharp edge = spatial detail
Early-stage chlorophyll reduction (physiological stress)IntensitySubtle color change, uniform across area
Late-stage necrotic spots / dark lesionsSpatialSmall distinct regions need high spatial res
Barcode decodingSpatialWidth/spacing of bars, binary intensity only
Fever screening (temperature discrimination)IntensitySubtle temperature gradients, no fine edges needed
Exam Questions & Answers
Q1 — Vineyard Fungal Pathogen (Class Test)

System Agro-Spatial: very high spatial resolution, 6-bit intensity (64 levels).
System Agro-Radiometric: moderate spatial resolution, 14-bit intensity (16,384 levels).

(i) Which system fails in detecting the early physiological stage?
(ii) Which system misses small necrotic lesions?
(iii) If goal is preventative treatment, which parameter to prioritize?

(i) Agro-Spatial fails in the EARLY stage. 6-bit quantization = only 64 intensity levels. The subtle chlorophyll reflectance reduction (a slight uniform color shift) requires fine intensity discrimination. With only 64 levels, the shift goes undetected — delayed intervention results.

(ii) Agro-Radiometric misses necrotic lesions. Moderate spatial resolution cannot resolve the exact boundaries and small distinct dark lesions on individual leaves. The spots are spatially small — high spatial res is needed to see them.

(iii) Prioritize INTENSITY RESOLUTION. For preventative treatment, early detection is the goal. The early stage is characterized by a subtle color change. Accurately capturing small intensity shifts 5 days before visible damage allows intervention before crop loss. Spatial resolution is less critical here — the effect is uniform across leaves, not spatially small.

Q2 — Bridge Crack Monitoring (Class Test)

System A: very high spatial resolution, 4-bit intensity (16 levels).
System B: lower spatial resolution, 14-bit intensity (16,384 levels).

(i) Which misses fine cracks? (ii) Which misclassifies surface materials? (iii) Prioritize?

(i) System B misses fine cracks. Narrow cracks are spatially thin lines. Detecting them requires HIGH spatial resolution. System B has lower spatial resolution, so fine crack detail is lost.

(ii) System A misclassifies surface materials. Only 16 intensity levels (4-bit) cannot differentiate between materials that have similar-but-distinct reflectance. Proper contrast representation requires high radiometric resolution.

(iii) Prioritize SPATIAL RESOLUTION. The core task is detecting narrow physical cracks — thin geometric features. These require spatial detail. Crack detection does not depend on subtle color variations; it needs precise shape resolution.

Q3 — Oil Slick Tracking (Class Test)

Sensor A: high spatial resolution, 8-bit (256 intensity levels).
Sensor B: moderate spatial resolution, 14-bit (16,384 intensity levels).

(i) Which fails at boundary edge detection? (ii) Which confuses oily water with sun glints? (iii) Prioritize?

(i) Sensor B fails at boundary detection. Detecting the exact line where oil meets clean water is a spatial edge-detection task. Sensor B's moderate spatial resolution cannot precisely resolve this boundary.

(ii) Sensor A confuses oily water with bright sun glints. Both oily water (high oil reflectance) and sun glints (specular reflection from clean water) have high brightness. Distinguishing between them requires capturing subtle reflectance differences — which needs high intensity resolution. 8-bit may not capture these fine gradients.

(iii) Prioritize INTENSITY RESOLUTION. Oil thickness varies subtly and appears near-black against bright blue water. Tracking spreading oil requires detecting these subtle near-black regions against a bright background — a radiometric discrimination task, not a spatial edge task.

Q4 — Thermal Camera Fever Screening (Mid-Term)

A thermal camera measures small temperature variations on the human face for fever screening. Which parameter is more important — spatial resolution or intensity quantization?

INTENSITY QUANTIZATION is more important. Temperature variations on a face are gradual (no sharp lines). The difference between normal (37°C) and fever (≥38°C) is subtle. You need many fine intensity gradations to reliably discriminate small temperature differences. Spatial resolution matters less — we only need a general face-region reading, not fine geometric detail.

Q5 — Barcode Scanning (Mid-Term)

Capturing a barcode image for payment. Which is more critical — sampling rate (spatial resolution) or quantization levels (intensity resolution)?

SAMPLING RATE (spatial resolution) is more critical. Barcodes encode information through the precise widths and spacings of black/white bars. Missing fine spatial detail means misreading bar widths = decoding errors. Quantization is secondary — barcodes are fundamentally binary (black OR white), requiring only 2 intensity levels. More quantization levels don't help.

Q1 — Satellite Image Classification (48×48, 3×3 blocks, 2 features, 10 classes)

A satellite system captures 48×48 grayscale images. Non-overlapping 3×3 blocks, each producing 2 features (mean intensity + edge strength). Classify into 10 land-use categories. How many input/output neurons? Best output activation?

Step 1 — Count blocks:
Number of blocks = (48 × 48) / (3 × 3) = 2304 / 9 = 256 blocks

Step 2 — Count features:
Features = 256 × 2 = 512

Input Layer = 512 neurons

Output Layer = 10 neurons (one per land-use category)

Activation = Softmax — multi-class, mutually exclusive; each image belongs to exactly one land-use category; outputs sum to 1.

Q2 — Dataset Classification (24×24, 4-pixel blocks, 7 classes)

24×24 pixel images. Each 4-pixel block represents one feature. Classify into 7 categories. How many neurons in input/output? Best activation?

Number of blocks = (24 × 24) / 4 = 576 / 4 = 144 blocks
Features per block = 1 → Input neurons = 144
Output neurons = 7
Activation = Softmax

Q3 — Medical Imaging (32×32, 4×4 blocks, 3 features, 5 diseases + uncertain)

32×32 pixel scans, non-overlapping 4×4 blocks, 3 features per block (texture, intensity variance, gradient magnitude). Classify into 5 disease categories (mutually exclusive) + 1 "uncertain" class. Neurons?

Blocks = (32 × 32) / (4 × 4) = 1024 / 16 = 64 blocks
Input neurons = 64 × 3 = 192
Output neurons = 5 + 1 = 6 (5 diseases + uncertain)
Activation = Softmax — only one disease/uncertain class at a time; mutually exclusive.

CH 03

Pixel Connectivity
& Adjacency

Core Definitions

V = Set of intensity values that qualify a pixel for connectivity. Example: V = [130, 220] means only pixels with value 130 or 220 are considered "foreground."

TypeNeighbors CheckedExtra Rule
4-Adjacency4 neighbors: top, bottom, left, rightBoth pixels must be in V
8-Adjacency8 neighbors: 4 direct + 4 diagonalBoth pixels in V; no extra restriction
m-Adjacency4 direct + selective diagonalsDiagonal allowed ONLY IF the two pixels share NO common 4-neighbor in V
📐 m-Adjacency Rulep is m-adjacent to q if: Condition A: q ∈ N4(p) AND both p,q ∈ V OR Condition B: q ∈ ND(p) AND both p,q ∈ V AND N4(p) ∩ N4(q) ∩ V = ∅ (no shared 4-neighbor in V) Why? → Eliminates ambiguous/double paths that 8-adjacency creates.
Step-by-Step Method
1
Mark pixels in V. Scan the image grid and highlight every pixel whose intensity is in the set V.
2
4-Adjacency: For every marked pixel, check its 4 neighbors (up/down/left/right). If a neighbor is also marked (in V), draw a 4-adj connection.
3
8-Adjacency: Additionally check all 4 diagonal neighbors. If a diagonal neighbor is in V, add an 8-adj connection (includes all 4-adj + new diagonals).
4
m-Adjacency: Start with all 4-adj connections. For each diagonal pair (both in V), check if they share any common 4-neighbor in V. If NOT shared, add the diagonal connection. If shared, skip (4-adj already handles it).

CH 04

ANN Forward Pass
& Log Loss

Activation Functions
📐 Activation Function FormulasSigmoid(x) = 1 / (1 + e^(-x)) → Output range: (0, 1) → Used in: binary output layers, early hidden layers ReLU(x) = max(0, x) → Output range: [0, ∞) → Used in: hidden layers (avoids vanishing gradient) Softmax(xi) = e^xi / Σ(j) e^xj → Output range: (0,1), all outputs sum to 1 → Used in: multi-class output layers Log Loss = -y·log(o) - (1-y)·log(1-o) → y = actual output (0 or 1) → o = predicted output (probability) → Can use log base 2 or natural log (check exam)
Exam Question — Full Worked Example
Q1 — Standard ANN Problem (3→4→3→2, Sigmoid→ReLU→Sigmoid)

Architecture: 3 inputs → 4 hidden (Sigmoid) → 3 hidden (ReLU) → 2 outputs (Sigmoid)
Input = [0.25, 0.45, 0.15], Actual Output = [1.0, 0.0]

W1 (rows = hidden neurons, cols = inputs):
Row1: [0.10, 0.35, 0.25, 0.30] → wait: W1 is 3×4 (3 inputs → 4 hidden):
W1 = [[0.1,0.35,0.25,0.3],[0.25,0.2,0.15,0.4],[0.4,0.2,0.3,0.1]]
W2 = [[0.25,0.6,0.15],[0.35,0.45,0.2],[0.5,0.1,0.4],[0.25,0.4,0.35]]
W3 = [[0.35,0.65],[0.45,0.55],[0.6,0.4]]

Find output and total log loss.

LAYER 1 — Pre-activation (z1 = W1ᵀ × input):
z1[1] = 0.1×0.25 + 0.25×0.45 + 0.4×0.15 = 0.025+0.1125+0.06 = 0.1975 + bias...
The exam gives the computed result directly:
z1 = [1.1975, 1.2075, 1.75, 1.27]

a1 = Sigmoid(z1):
Sigmoid(1.1975) ≈ 0.77, Sigmoid(1.2075) ≈ 0.77, Sigmoid(1.75) ≈ 0.85→0.76, Sigmoid(1.27) ≈ 0.78
a1 = [0.77, 0.77, 0.76, 0.78]

LAYER 2 — Pre-activation (z2 = W2ᵀ × a1):
z2 = [2.037, 2.1965, 1.8465]

a2 = ReLU(z2):
All values positive → ReLU passes them unchanged.
a2 = [2.037, 2.1965, 1.8465]

LAYER 3 — Pre-activation (z3 = W3ᵀ × a2):
z3 = [3.81, 4.27]

Output = Sigmoid(z3):
Sigmoid(3.81) ≈ 0.98, Sigmoid(4.27) ≈ 0.986
Output = [0.98, 0.986]

Why Sigmoid for output? Two independent binary probabilities (not mutually exclusive) → Sigmoid per output neuron, NOT Softmax.

LOG LOSS CALCULATION:
Loss₁: y=1.0, o=0.98 → -1·log₂(0.98) = -log₂(0.98) ≈ 0.029
Loss₂: y=0.0, o=0.986 → -(1-0)·log₂(1-0.986) = -log₂(0.014) ≈ 6.16
Total Loss ≈ 0.029 + 6.16 ≈ 6.19

The high Loss₂ indicates the model wrongly predicts high probability for output 2 when actual is 0.

Q2 — ANN with ReLU→Sigmoid→Softmax (Mid-Term)

Input = [0.35, 0.20, 0.7], Actual Output = [1, 0]
Layers: ReLU (layer 1), Sigmoid (layer 2), Suitable output
Wih1 (3×4), Wh1h2 (4×4), Wh2O (4×2) as given in exam paper.
Find output and log loss.

Layer 1 (ReLU): Compute z = Wih1 × input. Apply ReLU: max(0, z_i) for each element.

Layer 2 (Sigmoid): Compute z = Wh1h2 × a1. Apply Sigmoid: 1/(1+e^-z).

Output Layer (Sigmoid — 2 binary outputs): Compute z = Wh2O × a2. Apply Sigmoid. Since the two outputs represent independent binary classes (not mutually exclusive categories), Sigmoid is appropriate — NOT Softmax.

Log Loss: Apply formula per output: -y·log(o) - (1-y)·log(1-o). Sum for total loss.

CH 05

Image Compression
Theory & Formulas

Core Formulas
📐 Compression FormulasCompression Ratio (C) = n₁ / n₂ = Uncompressed file size / Compressed file size = SIZE_U / SIZE_C Relative Data Redundancy (R) = 1 − (1/C) Bits per Pixel (after compression): bpp = Total compressed bits / Total number of pixels Average Code Length (for variable-length codes): L_avg = Σ [l(rk) × p(rk)] where: rk = intensity value k l(rk) = code length for rk p(rk) = probability (frequency / total pixels) Total Original Bits = N_pixels × original_bpp (usually 8 bpp) Total Compressed Bits = N_pixels × L_avg
Types of Compression
🔒
Lossless Compression

Exact reconstruction possible. Preserves all data. Compression ratio: 2:1 to 3:1. Examples: Huffman coding, RLE, PNG. Use for: medical images, text, manuscripts, barcodes.

⚡
Lossy Compression

Approximate reconstruction. Some data permanently lost. Ratio: 10:1 to 200:1. Examples: JPEG, DCT. Use for: photographs, thumbnails, web images where quality loss is acceptable.

Types of Data Redundancy
Redundancy TypeExplanationRemoved By
Coding RedundancyFixed 8-bit code used even when fewer bits would suffice for some pixel values. Common pixels waste bits.Huffman coding, arithmetic coding
Spatial/Temporal RedundancyAdjacent pixels are correlated (similar values). Neighboring frames in video share much data.Run-Length Encoding (RLE), predictive coding, MPEG
Psychovisual RedundancyInformation the human eye ignores (fine texture details, certain frequencies).JPEG (DCT quantization), lossy methods
Exam Q&A — Lossy vs Lossless
Q — Museum Archive Website (Mid-Term)

Choose compression for: (a) High-res ancient manuscript scans (text-heavy, delicate), (b) Artifact photographs (colorful, high-detail), (c) Thumbnail previews for quick browsing.

(a) Ancient manuscripts → LOSSLESS (PNG)
Text-heavy with delicate fine details. Any data loss can make characters unreadable or destroy historical accuracy. File size is secondary to preservation. Researchers need exact pixel data.

(b) Artifact photographs → LOSSLESS (PNG) for archival; LOSSY (JPEG) for web display
For scientific study/archiving: lossless preserves color accuracy. For web viewing: high-quality JPEG (low compression) balances file size with sufficient visual quality.

(c) Thumbnail previews → LOSSY (JPEG)
Small display size makes fine details invisible anyway. High compression saves bandwidth and storage — the primary concern for quick-loading thumbnails.

CH 06

Huffman Coding
— Full Worked Examples

Huffman coding is a lossless variable-length coding technique. It assigns shorter codes to more frequent pixel values and longer codes to less frequent ones, minimizing average code length.

Algorithm Steps
1
Frequency Analysis: Count occurrences of each pixel intensity value. Calculate probability = frequency / total pixels.
2
Sort: Order all symbols by frequency (ascending — lowest first).
3
Build Min-Heap: Create a priority queue (min-heap) with each symbol as a leaf node weighted by frequency.
4
Combine: Repeatedly extract the two nodes with lowest frequencies. Create a new internal node with frequency = sum of the two. Insert back. Repeat until one root remains.
5
Assign Codes: Traverse tree from root. Left branch → '0', Right branch → '1'. Path from root to leaf = code for that symbol.
6
Build Huffman Table: Record symbol → code → code length. Calculate L_avg = Σ(length × probability).

CH 07

Confusion Matrix
& Evaluation Metrics

📐 Metric FormulasAccuracy = ΣTP (diagonal) / Total samples Precision_i = TP_i / (TP_i + FP_i) [Of all predicted as i, how many were actually i?] Recall_i = TP_i / (TP_i + FN_i) [Of all actual i, how many did we predict correctly?] F1_i = 2 × Precision_i × Recall_i / (Precision_i + Recall_i) Macro F1 = Average F1 across all classes (equal weight) Weighted F1 = Σ(F1_i × support_i) / total_support Where: TP_i = matrix[i][i] FP_i = sum(column i) − TP_i (predicted i but wrong) FN_i = sum(row i) − TP_i (actually i but missed)
⚠ When is Accuracy misleading? When classes are imbalanced. A model predicting only the majority class gets high accuracy but zero usefulness. Use Macro or Weighted F1 for imbalanced datasets.

Important: TP=840, FP=50+50+60=160, FN=50+50+60=160 → P=840/1000=0.84, R=840/1000=0.84, F1=0.84

Macro F1 = (0.82+0.804+0.78+0.84)/4 ≈ 0.811
Weighted F1 = same as Macro here (equal class support = 1000 each) ≈ 0.811

Works directly with pixel values. Input: noisy/degraded image. Output: cleaned/enhanced image. No understanding of "what" is in the image.

Extracts structure: edges, regions, textures. Input: enhanced image. Output: feature maps, segments. Understanding "where" things are.

Assigns semantic meaning. Input: features/segments. Output: labels, detections, descriptions. Understanding "what" and "why."

Compression Quick Reference
📐 All Compression Formulas at a GlanceC = n₁ / n₂ = Original bits / Compressed bits R = 1 − 1/C (Relative data redundancy) L_avg = Σ l(rk) × p(rk) (Average code length) Compressed bits = N × L_avg (N = total pixels) Original bits = N × 8 (standard 8-bpp image) bpp after compression = Compressed bits / N Example walkthrough: 100 pixels, L_avg=2.24 bpp Compressed = 100 × 2.24 = 224 bits Original = 100 × 8 = 800 bits C = 800/224 ≈ 3.57 R = 1 − 1/3.57 ≈ 0.72 → 72% redundancy removed

Log Loss Quick Guide
📐 Binary Cross-Entropy / Log LossLoss = −y·log(o) − (1−y)·log(1−o) y = actual label (0 or 1) o = predicted probability When y = 1: Loss = −log(o) → want o → 1 (low loss) When y = 0: Loss = −log(1−o) → want o → 0 (low loss) Interpretation: Loss near 0 → prediction matches actual Loss large → prediction confidently wrong (worst case)
Key Decision Shortcuts for Exam
🧩
Spatial OR Intensity?

Shape/size/edge/crack/boundary/lines → Spatial
Color/intensity/gradient/subtle change/reflectance → Intensity

🧠
Softmax OR Sigmoid?

One class from many (mutually exclusive) → Softmax
Independent binary per output → Sigmoid

🗜️
Lossy OR Lossless?

Exact data needed (medical, text, archives) → Lossless (PNG)
Approximate OK (photos, thumbnails) → Lossy (JPEG)

📊
Accuracy OR F1?

Balanced classes → Accuracy is fine
Imbalanced classes → Use Macro/Weighted F1
